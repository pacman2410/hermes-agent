# User Memory Architecture

Purpose: make the Digital Career Strategist more useful over time by preserving user context, progress, evidence, and recommendations.

## Memory Categories

Goal Memory:

- Primary goal
- Secondary goals
- Target role
- Target salary
- Timeline
- Motivation
- Constraints

Assessment History:

- Assessment type
- Date
- Overall score
- Component scores
- Score trend
- Interpretation

Blueprint History:

- Current state
- Desired state
- Milestones
- Action plan
- Risks
- Dependencies
- Success metrics
- Version history

Weekly Check-Ins:

- Completed actions
- Missed actions
- Blockers
- Confidence
- Momentum
- Evidence captured
- Recommended next action

Career History:

- Roles
- Industries
- Skills
- Accomplishments
- Transitions
- Education or certifications

Promotion Evidence:

- Accomplishments
- Metrics
- Stakeholder feedback
- Projects
- Recognition
- Leadership examples

Conversation History:

- User questions
- AI recommendations
- Human reviews
- Scripts generated
- Decisions made

Recommendation History:

- Recommendation
- Source framework
- Accepted or ignored
- Completion status
- Outcome

Learning History:

- Skills selected
- Practice actions
- Resources used
- Evidence of application

## Memory Retrieval Rules

Retrieve memory based on:

- Current user intent
- Active blueprint
- Relevant score component
- Recent check-ins
- High-stakes timing
- Similar past conversations

Do not retrieve irrelevant old context unless it changes the recommendation.

## Memory Update Rules

Update memory when:

- User completes assessment.
- User changes goal.
- User completes check-in.
- User adds evidence.
- User books human review.
- User reports career outcome.
- Coach adds note.

## Privacy And Control

Users should eventually be able to:

- View stored career memory.
- Edit incorrect memory.
- Delete sensitive memory.
- Disable use of specific memory categories.
- Export their career blueprint and evidence.

## Personalization Logic

The system should personalize by:

- Career stage
- Target outcome
- Score weaknesses
- Work context
- Confidence level
- Action completion pattern
- Communication preference
- Risk profile

Example:

If user has high results but low visibility, recommend stakeholder visibility actions instead of more skill-building.

If user has low confidence and low completion, recommend smaller next actions and human review only if stakes are high.
